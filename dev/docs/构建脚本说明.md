# XGE 构建脚本说明

本文档记录 XGE V2 当前阶段的构建脚本约定。项目暂不使用 CMake，优先保留轻量、可读、可直接调试的脚本。

## 基本约定

- Windows 使用 `*.bat`。
- Linux/macOS/类 Unix 使用 `*.sh`。
- 根目录只保留跨模块或批量入口；单个示例的构建脚本放在 `examples/<name>/build.bat` / `examples/<name>/build.sh`。
- 编译输出统一放入 `build/` 目录。
- 引擎源码入口统一为根目录 `xge.c`。
- 公开头为根目录 `xge.h`。
- 示例源码放在 `examples/<name>/main.c`。
- 测试源码放在 `test/`。
- 第三方依赖放在 `lib/`。

## 环境要求

Windows:

- 需要 `gcc` 位于 `PATH` 中。
- 当前脚本使用 MinGW 风格链接参数。
- 默认链接库包括 `ws2_32`、`iphlpapi`、`gdi32`、`user32`、`shell32`、`opengl32`、`ole32`、`winmm`、`avrt`。

Linux/macOS/类 Unix:

- 需要 `cc` 可用。
- 默认链接 `-lm -ldl -pthread`。
- 平台图形依赖会随着 Sokol/EGL/平台后端完善继续补充。

## 核心脚本

- `build_test.*`：编译并运行纯 C 测试程序。
- `build_dll.*`：编译动态库输出。
- `build_examples_all.bat`：Windows 下编译所有可本地构建的示例；`--xui` 只编译 XUI 验证示例。
- `run_examples_smoke.bat`：构建并运行支持自动退出的示例 smoke；`--xui [frames]` 会先构建 XUI 示例并只运行 XUI smoke，用于避开非 XUI 阻断项。
- `examples/core_lifecycle/build.*`：编译 Core 生命周期示例，覆盖手动帧、窗口主循环和 frame stats。
- `examples/diagnostics_caps_log/build.*`：编译诊断示例，覆盖日志、平台能力、GPU caps 和 runtime counters。
- `examples/viewport_clip_camera/build.*`：编译 viewport、clip、camera 和坐标转换示例。
- `examples/image_texture_lab/build.*`：编译 image/texture lab，覆盖图像加载、纹理更新、采样、fallback 和 readback。
- `examples/font_text_lab/build.*`：编译 font/text lab，覆盖 TTF/TTC、XRF、fallback、glyph 和文本绘制。
- `examples/resource_provider_lab/build.*`：编译 resource provider lab，覆盖 mem provider、mock xpack 和 resource/texture/font 加载。
- `examples/input_state_lab/build.*`：编译 input state lab，覆盖键盘、鼠标、剪贴板、触摸和模拟手柄。
- `examples/audio_lab/build.*`：编译 audio lab，覆盖 sound/music/stream、group、listener、fade 和 fallback。
- `examples/scene_lifecycle_lab/build.*`：编译 scene lifecycle lab，覆盖 scene 栈、生命周期回调和更新策略。
- `examples/auto_pause_lab/build.*`：编译 auto pause lab，覆盖 scene pause/resume、freeze snapshot 和 pause overlay。
- `examples/render_target_lab/build.*`：编译 render target lab，覆盖 render target、pass、texture 和 readback。
- `examples/shader_variant_lab/build.*`：编译 shader variant lab，覆盖 shader 创建、uniform、addref 和 variant define 缓存。
- `examples/material_mesh_buffer_lab/build.*`：编译 material mesh buffer lab，覆盖 buffer、mesh 和 material 绘制路径。
- `examples/blend_depth_color_lab/build.*`：编译 blend depth color lab，覆盖 color、blend 和 depth test 状态。
- `examples/shape_full_gallery/build.*`：编译 shape full gallery，覆盖完整 shape API、auto batch 和 shape batch。
- `examples/async_assets_lab/build.*`：编译 async assets lab，覆盖 async request、threading、poll 和 image/texture/font/sound 异步加载。
- `examples/platform_runtime_lab/build.*`：编译 platform runtime lab，覆盖 runtime counters、高 DPI 和平台 smoke 面板。
- `examples/offscreen_egl_lab/build.*`：编译 offscreen EGL lab，覆盖 EGL capability、offscreen target、fallback 和 readback。
- `examples/render_thread_lab/build.*`：编译 render thread lab，覆盖 render thread caps、worker drain、EGL-owned context 和当前 window context 限制。
- `examples/miniprogram_bridge_lab/build.*`：编译 miniprogram bridge lab，覆盖小程序宿主 bridge、touch/text、request-frame、audio command 和 `res://` provider。
- `examples/xui_widget_tree_style_lab/build.*`：编译 XUI widget tree style lab，覆盖 widget tree、style、rect/content rect 和 mark/refresh 自检。
- `examples/xui_layout_gallery/build.*`：编译 XUI layout gallery，覆盖 absolute/row/column/stack/grid、size unit 和 dirty layout 自检。
- `examples/xui_focus_capture_event_lab/build.*`：编译 XUI focus capture event lab，覆盖 hit-test、event queue、focus/capture、pointer、keynav 和 context 事件流。
- `examples/xui_paint_host_lab/build.*`：编译 XUI paint host lab，覆盖自定义 host、paint callback、paint command 统计、clip 和 refresh request。
- `examples/xui_theme_lab/build.*`：编译 XUI theme lab，覆盖 default theme、theme set/get、DIP scale、styleFromTheme 和 light/dark 切换。
- `examples/xui_basic_controls_lab/build.*`：编译 XUI basic controls lab，覆盖 Label/Image/Button icon mode/Panel/Separator 的 init/unit/setter 和 state 自检。
- `examples/xui_choice_controls_lab/build.*`：编译 XUI choice controls lab，覆盖 Toggle/CheckBox/RadioGroup/Radio 的 getter、state 和 keyboard/mouse 交互自检。
- `examples/xui_list_scroll_lab/build.*`：编译 XUI list scroll lab，覆盖 ScrollView/ListView、disabled rows、hover、PageUp/PageDown 和 scroll getter/setter 自检。
- `examples/mvp/build.*`：编译最小 MVP 示例程序。
- `examples/platform_smoke/build.*`：编译通用平台冒烟示例，输出 caps 并绘制基础 shape/texture。
- `run_platform_smoke.*`：运行通用 platform smoke 示例，自动退出并检查日志。
- `check_platform_smoke.bat`：Windows 下构建、运行并检查通用 platform smoke。
- `check_platform_backends.*`：检查当前机器是否具备常见平台后端工具链。
- `check_platform_scaffold.*`：检查 Android/iOS/Web/Linux/macOS 平台 scaffold 文件和关键配置是否完整。
- `check_platform_results.*`：读取 `dev/docs/平台后端验证结果.md`，汇总哪些平台任务可关闭。
- `check_platform_smoke_log.*`：检查 platform smoke 输出日志是否包含 caps 和 runtime 快照。
- `check_linux_x11_smoke.sh`：Linux X11 下构建、运行并检查 Sokol/GLCore platform smoke。
- `check_linux_egl_smoke.sh`：Linux X11/EGL 下构建、运行并检查 Sokol/GLES3 platform smoke。
- `check_macos_smoke.sh`：macOS 下构建、运行并检查 Sokol/OpenGL platform smoke。
- `check_platform_all.*`：串行执行 scaffold、工具链和结果状态检查。
- `check_script_docs.*`：检查文档中引用的脚本是否真实存在。
- `check_docs.*`：检查正式文档链接，计划中文档只给警告，真实断链会失败。
- `record_platform_result.*`：把目标平台冒烟结果写入 `dev/docs/平台后端验证结果.md`。
- `examples/egl_surfaceless/build.sh`：在目标 Linux 板卡上编译 EGL surfaceless 冒烟示例。
- `check_board_linux_egl.sh`：在目标 Linux 板卡上顺序构建并运行 pbuffer 与 surfaceless EGL 冒烟示例。
- `examples/platform_smoke/build_macos.sh`：在 macOS 上使用 Sokol + OpenGL 编译 MVP 示例。
- `build_android_ndk.*`：使用 Android NDK/CMake 编译 NativeActivity scaffold。
- `build_android_apk.*`：使用 `platform/android/gradle` Gradle scaffold 编译 Android debug APK。
- `run_android_apk.*`：使用 `adb` 安装并启动 Android debug APK。
- `build_ios_sim_exe.sh`：在 macOS 上使用 Xcode 命令行工具编译 iOS simulator MVP app bundle。
- `run_ios_sim_exe.sh`：使用 `xcrun simctl` 安装并启动 iOS simulator app bundle。
- `examples/platform_smoke/build_linux_x11.sh`：在 Linux X11 上使用 Sokol + OpenGL 编译 platform smoke 示例。
- `examples/platform_smoke/build_linux_egl.sh`：在 Linux X11 上使用 Sokol + EGL/GLES3 编译 MVP 示例。
- `examples/platform_smoke/build_web.*`：使用 Emscripten 编译 Web/WebGL2 MVP 示例，并使用 `platform/web/shell.html` 作为浏览器 shell。
- `serve_web_exe.*`：启动本地 HTTP server 运行 Web/Emscripten 构建产物。
- `examples/offscreen/build_board_linux_egl.sh`：在目标 Linux 板卡上编译 EGL pbuffer 冒烟示例。

常用本地验证顺序：

```bat
build_test.bat
build_dll.bat
examples\mvp\build.bat
```

## 新增脚本规则

新增示例时优先在示例目录内添加 Windows 与 Unix 两个脚本：

- `examples/<example>/build.bat`
- `examples/<example>/build.sh`

脚本内容应保持简单：

- 设置 `OUT_DIR=build`。
- 设置 `OUT` 为 `build/xge_<example>` 或 `build\xge_<example>.exe`。
- 设置 `SRC` 为 `xge.c examples/<example>/main.c`。
- 设置 `INC=-I.`。
- 使用项目统一 warning 参数。
- 编译失败时返回非 0。

## 单头文件说明

仓库源码按模块拆分，便于阅读和维护。单头文件版本由 `singlehead/` 工具生成，不应把大量实现直接写回 `xge.h`。

后续单头文件生成器完善后，构建脚本应继续以源码拆分版为开发主路径，单头文件测试只验证集成产物。

## 平台验证结果

跨平台冒烟的实际执行结果统一记录到 `docs/平台后端验证结果.md`。脚本新增或验证流程变化时，应同步更新该文件中的执行脚本列表。

可用以下脚本检查当前验证结果是否足以关闭平台 spec 项：

```bat
check_platform_results.bat
```

Linux/macOS:

```sh
./check_platform_results.sh
```

目标平台执行完成后，可以用以下脚本记录结果：

```bat
record_platform_result.bat "Windows" 通过 --env "Windows 11 / GPU / gcc" --scripts "build_test.bat; examples\platform_smoke\build.bat" --summary "窗口和基础渲染正常"
```

Linux/macOS:

```sh
./record_platform_result.sh "Linux X11" 通过 --env "Ubuntu / X11 / Mesa" --scripts "./examples/platform_smoke/build_linux_x11.sh" --summary "窗口、OpenGL、输入正常"
```

platform smoke 示例支持自动退出参数，便于在目标机采集日志：

```sh
./build/xge_linux_x11 --frames 180 > smoke.log
./check_platform_smoke_log.sh smoke.log --require-final
```

Windows:

```bat
build\xge_platform_smoke.exe --frames 180 > smoke.log
check_platform_smoke_log.bat smoke.log --require-final
```

也可以直接使用包装脚本：

```bat
check_platform_smoke.bat
```

Linux/macOS:

```sh
./check_linux_x11_smoke.sh
./check_linux_egl_smoke.sh
./check_macos_smoke.sh
```
