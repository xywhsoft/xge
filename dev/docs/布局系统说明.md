# XUI 布局系统说明

本文档固化 XUI 孵化期布局系统的设计边界。布局系统归属 XUI，不进入 XGE 内核；当前位于 XGE 仓库内只是为了调试渲染、输入、字体、dirty rect 和 RMGUI 工作流，成熟后应剥离到 XUI 仓库。

## 设计目标

XUI 布局系统服务于 DUI/RMGUI，而不是复刻 HTML/CSS。

目标：

- 足够表达游戏 UI、工具 UI、APP 风格面板和常规控件布局。
- 性能可预测，布局过程尽量单次递归完成。
- API 简单，结构透明，便于 C 项目使用。
- 支持 DIP、百分比、内容尺寸和 grow。
- 支持 dirty layout / dirty paint，避免每帧全树重算。
- 支持批量修改，避免连续属性更新造成多次刷新请求。

非目标：

- 不实现完整 CSS cascade。
- 不实现复杂 flexbox/grid 规范。
- 不实现浮动布局、表格布局、多列排版。
- 不在第一版内置复杂富文本排版。
- 不把布局逻辑耦合到 GL、Sokol 或 XGE core。

## 当前布局类型

第一版固定以下布局类型：

```c
#define XGE_XUI_LAYOUT_ABSOLUTE	0
#define XGE_XUI_LAYOUT_ROW		1
#define XGE_XUI_LAYOUT_COLUMN	2
#define XGE_XUI_LAYOUT_STACK	3
#define XGE_XUI_LAYOUT_GRID		4
```

### Absolute

Absolute 是默认布局，用于：

- 手动定位控件。
- 游戏 HUD。
- 编辑器 overlay。
- 需要 anchor 的窗口或面板。

子元素可以通过 `xgeXuiWidgetSetRect` 直接设置 rect，也可以通过 size、align、anchor 参与父容器内定位。

### Row

Row 按水平方向排列子元素。

适合：

- 工具栏。
- 表单行。
- 图标 + 文本。
- 左右区域分配。

规则：

- 子元素按添加顺序排列。
- 主轴为 X，交叉轴为 Y。
- margin 参与占位。
- grow 子元素分配剩余宽度。
- align 控制交叉轴位置或 stretch。

### Column

Column 按垂直方向排列子元素。

适合：

- 属性面板。
- 列表项内部布局。
- 对话框内容区。
- 设置页。

规则与 Row 对称：

- 主轴为 Y。
- grow 子元素分配剩余高度。
- align 控制 X 方向。

### Overflow

XUI 第一版不把 Row/Column 做成自动滚动容器，避免引入完整 CSS overflow 模型和额外热路径状态。空间不足时，Row/Column 仍按布局结果放置子元素；最终显示和命中由容器裁剪策略决定。

语义：

- `visible`：默认行为，不启用 `XGE_XUI_WIDGET_CLIP`。子节点可以溢出显示，并按自身计算后的 rect 参与命中。
- `clip`：调用 `xgeXuiWidgetSetClip(widget, 1)` 或控件封装的 clip API。绘制裁剪到 widget 的 `tContentRect`，命中测试也不会继续进入 `tContentRect` 外的子节点。
- `scroll`：使用 `xgeXuiScrollView` 或 `xgeXuiListView`。滚动容器自己维护内容尺寸、滚动偏移和滚动条，并默认启用 clip。

约束：

- Row/Column 不自动根据 overflow 创建滚动条。
- `clip` 裁剪的是 content rect，不是完整 widget rect；padding 区域仍属于 widget 自身，但不再命中溢出的子节点。
- 需要滚动的 APP 页面或游戏面板应显式使用 ScrollView/ListView，以保持行为和内存开销可控。

### Stack

Stack 把子元素叠放在父容器内容区内。

适合：

- 背景 + 内容 + overlay。
- tab 页面。
- modal shadow。
- 自定义控件内部图层。

规则：

- 每个子元素都基于父 content rect 计算。
- 子元素可以使用 percent、grow、align、anchor。
- 绘制顺序仍按 widget tree 和 z 规则决定。

### Grid

Grid 是轻量网格，不是 CSS Grid。

当前 API：

```c
void xgeXuiWidgetSetGrid(xge_xui_widget pWidget, int iColumns, float fRowHeight, float fColumnGap, float fRowGap);
void xgeXuiWidgetSetGridColumnSpan(xge_xui_widget pWidget, int iColumnSpan);
```

适合：

- 固定列数的图标面板。
- 简单表单。
- 背包格子。
- 工具按钮组。

规则：

- 固定列数。
- 行高由 `fRowHeight` 指定，小于等于 0 时使用单元格宽度，形成方格网格。
- 列宽由容器宽度、padding、gap 和列数计算。
- 子元素默认占 1 列；`xgeXuiWidgetSetGridColumnSpan` 可让子元素横跨多列，超出剩余列时自动换到下一行。
- 第一版只支持 column span，不支持 row span、命名网格区或占用矩阵回填。

## 尺寸单位

当前尺寸 API：

```c
xge_xui_size_t xgeXuiSizePx(float fValue);
xge_xui_size_t xgeXuiSizeDip(float fValue);
xge_xui_size_t xgeXuiSizePercent(float fValue);
xge_xui_size_t xgeXuiSizeGrow(float fValue);
xge_xui_size_t xgeXuiSizeContent(void);
```

单位语义：

- PX：物理像素或 XGE 当前 UI 像素单位。
- DIP：设备无关像素，乘以 context 的 `dipScale`。
- Percent：相对父容器对应轴尺寸。
- Grow：参与 row/column/grid 剩余空间分配。
- Content：使用控件测量结果。

约定：

- `xgeXuiSetDipScale(context, scale)` 用于 DPI 缩放。
- scale 小于等于 0 时回退到 1。
- DIP 变化会标记 layout 和 paint dirty。
- Content 依赖控件测量，常见来源为 text、image、custom measure。

## 盒模型

XUI 使用简化盒模型：

```text
widget rect
  margin
  border/radius visual only
  padding
  content rect
```

当前重点：

- margin 影响父布局对子元素的占位。
- padding 影响父容器的 content rect。
- radius 只影响绘制，不影响布局。
- background 只影响绘制，不影响布局。
- border 第一阶段不作为独立布局厚度处理；如果后续加入 border width，应明确是否进入 content rect。

公开 API：

```c
void xgeXuiWidgetSetMarginPx(xge_xui_widget pWidget, float fLeft, float fTop, float fRight, float fBottom);
void xgeXuiWidgetSetPaddingPx(xge_xui_widget pWidget, float fLeft, float fTop, float fRight, float fBottom);
```

后续可以补充 DIP/percent edge API，但第一阶段 PX 足够覆盖控件和编辑器需求。

## Root 与 Safe Area

XUI root 和 overlay root 会在 `xgeXuiUpdate` 时自动跟随窗口尺寸。业务界面需要避开系统栏、刘海或统一外边距时，使用 `xgeXuiSetSafeAreaPx` 设置 root padding；root 的 content rect 会避开该区域，overlay root 仍保持全窗口尺寸，方便弹窗、菜单和调试层覆盖全屏。

## 对齐与 Anchor

对齐：

```c
#define XGE_XUI_ALIGN_START		0
#define XGE_XUI_ALIGN_CENTER	1
#define XGE_XUI_ALIGN_END		2
#define XGE_XUI_ALIGN_STRETCH	3
```

API：

```c
void xgeXuiWidgetSetAlign(xge_xui_widget pWidget, int iAlignX, int iAlignY);
```

语义：

- START：靠左或靠上。
- CENTER：居中。
- END：靠右或靠下。
- STRETCH：填充可用空间。

Anchor：

```c
void xgeXuiWidgetSetAnchorPx(xge_xui_widget pWidget, int iAnchor, float fLeft, float fTop, float fRight, float fBottom);
int xgeXuiWidgetGetAnchor(xge_xui_widget pWidget);
```

Anchor 适合 Absolute/Stack 场景下把控件绑定到父容器边缘。四边 anchor 同时设置时，控件尺寸由父容器尺寸和边距推导。

## Min/Max

当前 API：

```c
void xgeXuiWidgetSetMinSize(xge_xui_widget pWidget, xge_xui_size_t tWidth, xge_xui_size_t tHeight);
void xgeXuiWidgetSetMaxSize(xge_xui_widget pWidget, xge_xui_size_t tWidth, xge_xui_size_t tHeight);
```

约定：

- min 默认 0。
- max 为 0 表示不限制。
- min/max 在 resolved size 后执行 clamp。
- min 大于 max 的异常情况后续应统一规范；第一阶段建议调用方避免传入矛盾约束。

## Dirty Layout 与 Dirty Paint

XUI 使用 retained tree，因此必须避免每帧重算全量布局。

状态：

- `XGE_XUI_WIDGET_DIRTY_LAYOUT`
- `XGE_XUI_WIDGET_DIRTY_PAINT`

规则：

- 改变布局属性时，标记当前 widget 到 root 的 layout dirty。
- 改变视觉属性时，标记 paint dirty。
- layout dirty 通常也意味着 paint dirty。
- `xgeXuiUpdate` 负责刷新 layout。
- `xgeXuiPaint` 负责生成或提交 paint command。

统计字段用于测试和调试：

- `iDirtyLayoutCount`
- `iDirtyPaintCount`
- `iPaintCommandCount`
- `iPaintFlushCount`

## 批量修改

当前 API：

```c
void xgeXuiLayoutBatchBegin(xge_xui_context pContext);
void xgeXuiLayoutBatchEnd(xge_xui_context pContext);
```

用途：

- 一次性修改多个控件属性。
- 减少重复 dirty 传播。
- 合并刷新请求。
- APP/RMGUI 中减少不必要的手动重绘。

规则：

- batch depth 可以嵌套。
- batch 内记录 dirty layout/paint 标志。
- 最外层 `End` 时统一标记 root dirty 并请求 refresh。

## 布局执行流程

推荐流程：

```text
xgeXuiUpdate
  -> update root size from host/window
  -> set active host and dipScale
  -> layout root
    -> resolve padding/content rect
    -> measure children if needed
    -> apply layout algorithm
    -> recurse children
  -> clear layout dirty
```

绘制流程：

```text
xgeXuiPaint
  -> traverse visible tree
  -> apply clip
  -> emit background/text/image/custom commands
  -> submit through host backend
  -> clear paint dirty / refresh flag
```

布局层不能直接调用 GL。绘制也应通过 host/backend 接口进入 XGE，方便未来剥离到 XUI 独立仓库。

## 性能约束

第一版性能原则：

- 布局算法保持 O(N)，N 为可见 widget 数。
- 不做全局约束求解。
- 不做多轮 shrink/grow 复杂回流。
- 不支持文本自动复杂换行参与父级多轮布局；简单测量可以使用 host measure。
- 不在布局阶段分配大量临时内存。
- 不在每帧无 dirty 时重算整棵树。

潜在优化：

- 子树 dirty flag。
- 可见性剪枝。
- 虚拟列表。
- paint command 缓存。
- text measure cache。
- widget pool。

## 与 XGE 的边界

最终依赖方向：

```text
xge -> xrt
xui_core -> xrt
xui_backend_xge -> xui_core + xge
```

当前孵化期允许 `src/xge_xui_*` 存在于 XGE 仓库，但代码必须保持可剥离：

- layout/widget/event 不直接依赖 GL。
- layout/widget/event 不依赖 Sokol。
- XGE 绘制调用集中在 host/backend。
- API 命名后续剥离时可以从 `xgeXui*` 迁移到 `xui*`，但语义保持稳定。
