# XGE 人工冒烟测试流程

本文档用于集中人工验证当前 XGE 第一阶段 MVP。自动测试只能覆盖 API 行为和无窗口路径，窗口显示、输入法、音频设备、GPU 驱动、小程序和板卡 Linux 仍需要人工观察。

## 测试前准备

- 在 Windows 上确认 `gcc` 位于 `PATH`。
- 从仓库根目录执行脚本。
- 每个示例启动后观察 10 到 30 秒，确认没有崩溃、黑屏、明显卡死或异常闪烁。
- 如果示例需要交互，至少执行一次对应输入。
- 如果某项因为本机缺少设备无法验证，在反馈中标记为 `跳过`，不要视为失败。

## Windows 手工综合验证

当前 Windows 收敛优先使用综合验证程序覆盖窗口、渲染、输入、IME、gamepad 模拟和压力图层：

```bat
build_manual_validation_exe.bat
build\xge_manual_validation.exe --stress 20000 --novsync
```

通过标准：

- 顶部仪表盘文字清晰可见，压力背景不会遮挡文字、准星或三块视觉测试区。
- 棋盘纹理、shape 面板、移动圆、鼠标准星均显示正常。
- 鼠标移动会更新 `mouse` 与 `delta`；左键、右键、滚轮计数会增加。
- 左键和空格会触发左侧蓝色矩形弹动；滚轮会触发黄色圆变大后回落。
- 英文输入和中文 IME 上屏会增加 `text events`，`last U+` 会更新。
- 按 `G` 可切换 `gamepad0 connected`，连接状态行应变为高亮绿色。
- 窗口缩放、最小化/恢复、运行 3 到 5 分钟无崩溃、黑屏、残影或明显卡死。
- 按 `ESC` 正常退出，并打印 `manual summary`。

压力基线参考：

```bat
build\xge_manual_validation.exe --bench 10 --stress 20000 --novsync
```

当前 Windows + AMD Radeon RX 6600 基线：`stress=20000` 基本流畅，`draw` 约 130，`batch` 约 6。该数值用于回归对比，不作为其他 GPU 的硬性门槛。

## 第一组：构建与核心回归

```bat
build_test.bat
build_dll.bat
build_exe.bat
```

通过标准：

- `build_test.bat` 退出码为 0。
- `build_dll.bat` 能生成 DLL。
- `build_exe.bat` 能生成并启动最小窗口。
- 最小窗口能清屏、刷新、正常关闭。

## 第二组：基础绘制

```bat
build_shape_exe.bat
build_texture_exe.bat
build_text_exe.bat
build_sprite_batch_exe.bat
```

通过标准：

- shape 示例能看到点、线、矩形、圆、弧线、三角形或多边形。
- texture 示例能加载并显示纹理。
- text 示例能显示英文和中文文本，文本没有明显错位。
- sprite batch 示例能显示多 sprite，窗口运行期间无崩溃。

## 第三组：渲染管线

- perspective quad 示例能看到透视或 2.5D 变形。

## 第五组：资源、音频、调试与 XUI

```bat
build_async_exe.bat

build_xui_layout_validation_exe.bat

```

也可以先用聚合脚本一次性构建所有 XUI 验证程序。日常优先使用并行脚本，若 PowerShell job 环境异常，再使用串行脚本兜底：

```bat
build_xui_validation_all_parallel.bat
build_xui_validation_all_exe.bat
```

构建完成后，也可以用以下脚本按顺序启动 XUI 验证窗口。每关闭一个窗口，会自动进入下一个验证程序：

```bat
run_xui_validation_manual.bat
```

通过标准：

- async 示例能显示 fallback 资源并在加载完成后切换到目标资源。
- audio 示例能播放 sound/music，音量、循环或 3D 位置接口没有崩溃。
- debug overlay 示例能显示帧统计或调试信息。
- XUI incubation 示例能显示基础控件。
- XUI bridge 示例能把输入事件传递到 XUI，并能触发控件状态变化。
- XUI layout validation 示例能显示 row/column/grow/content 布局，窗口缩放后没有明显错位、漂移或闪烁。
- XUI input validation 示例能验证单行编辑框的输入、选择、剪贴板、IME 和只读/禁用状态。
- XUI text edit validation 示例能验证多行编辑框的换行、光标移动、滚动、剪贴板和 IME。

### XUI incubation 控件验证

```bat
build_xui_incubation_exe.bat
build\xge_xui_incubation.exe
```

该示例作为当前核心 XUI 控件的最小综合示例，覆盖 `Panel`、`Button`、`IconButton`、`Tooltip`、`Tabs`、`Progress`、`CheckBox`、`RadioButton`、`RadioGroup`、`Switch`、`ComboBox`、`Separator`、`Splitter`、`ScrollBar`、`ScrollView` 和 `Popup`。

通过标准：

- 窗口启动后能看到左侧 `XUI Incubation` 面板、右侧滚动区域、垂直滚动条和中间分割条。
- `Button` 默认显示 `Open Popup`，鼠标悬停、按下、移出后颜色能在 normal/hover/active 间正确恢复。
- 点击 `Button` 后弹出 popup，按钮文字变为 `Close Popup`；再次点击、点击 popup 外部或按 `ESC` 后 popup 关闭。
- `IconButton` 能显示图标，悬停后显示 `Icon action` tooltip，移出后 tooltip 消失；点击后退出时 `icon_clicks` 计数增加。
- 点击 `Main`、`State`、`Input` 标签，选中态随点击移动，退出日志中的 `tab` 和 `tab_changes` 会更新。
- `Progress xx%` 文本和绿色进度条可见，并随时间循环增长。
- 点击 `CheckBox` 可切换勾选状态，退出日志中的 `checkbox` 和 `checkbox_changes` 会更新。
- 点击 `Radio A`、`Radio B` 时同组互斥，退出日志中的 `radio` 和 `radio_changes` 会更新。
- 点击 `Switch` 可切换开关状态，退出日志中的 `switch` 和 `switch_changes` 会更新。
- 点击 `ComboBox` 可展开下拉列表，选择 `Small`、`Medium`、`Large` 或 `Custom` 后列表关闭，退出日志中的 `combo` 和 `combo_changes` 会更新。
- 拖动中间 `Splitter` 后分割条跟随鼠标移动，退出日志中的 `splitter` 和 `splitter_changes` 会更新。
- 拖动右侧独立 `ScrollBar` 的 thumb，或点击 track，滚动值变化，退出日志中的 `scrollbar` 和 `scrollbar_changes` 会更新。
- 右侧 `ScrollView` 的内容和滚动条能正常绘制，自动滚动过程中没有残影、错色或闪烁。
- 按 `ESC` 能退出程序；若 popup 正在打开，第一次 `ESC` 关闭 popup，第二次 `ESC` 退出。

### XUI layout validation 布局验证

```bat
build_xui_layout_validation_exe.bat

```

通过标准：

- 按 `ESC` 正常退出。

### XUI input validation 编辑框验证

```bat
build_xui_input_validation_exe.bat
build\xge_xui_input_validation.exe
```

通过标准：

- 窗口启动后能看到 `normal`、`password`、`readonly`、`disabled` 四个输入框。
- `normal` 输入框为空时显示 placeholder；输入英文和中文 IME 后文本正常显示，光标会闪烁。
- 在 `normal` 中输入长文本，光标移动到右侧时文本会横向滚动，Home/End 能跳到开头/结尾。
- `Ctrl+Left`、`Ctrl+Right` 能按词移动；`Ctrl+A` 能全选；双击英文单词能选中词。
- `Ctrl+C`、`Ctrl+X`、`Ctrl+V` 能完成复制、剪切、粘贴。
- `password` 输入框显示掩码字符，退出日志中的 `password_len` 反映真实长度。
- `readonly` 输入框可以获得焦点和选择文本，但输入、删除、剪切、粘贴不会修改内容。
- `disabled` 输入框不能获得输入焦点，也不会响应编辑操作。
- 按 `ESC` 正常退出，并打印 `xui input summary`。

### XUI text edit validation 多行编辑框验证

```bat
build_xui_text_edit_validation_exe.bat
build\xge_xui_text_edit_validation.exe
```

通过标准：

- 窗口启动后能看到一个大号多行文本区，初始文本包含多行内容和一条长行。
- 点击文本区任意位置，光标应移动到接近点击位置，并开始闪烁。
- 输入英文和中文 IME 能插入到当前光标位置。
- 按 `Enter` 能插入新行，`Backspace` 和 `Delete` 能删除字符或选区。
- `Left/Right/Up/Down`、`Home/End`、`Ctrl+Home/Ctrl+End`、`PageUp/PageDown` 能移动光标。
- 长行和多行文本中，光标移动到不可见位置时，文本区能自动水平或垂直滚动到光标附近。
- `Ctrl+A` 能全选，`Ctrl+C`、`Ctrl+X`、`Ctrl+V` 能复制、剪切、粘贴。
- 文本插入、删除、剪切、粘贴后，`Ctrl+Z` 能撤销，`Ctrl+Y` 能重做。
- 按 `Ctrl+W` 能切换自动换行；长行应在文本区内换成多行显示，再按一次恢复横向滚动模式。
- 鼠标拖拽可以改变选择范围；当前阶段如果选择高亮不明显，应记录为后续视觉改进项。
- 在较多行文本中反复移动光标、PageUp/PageDown 和拖选时，不应出现明显卡顿或光标定位错乱。
- 按 `ESC` 正常退出，并打印 `xui textedit summary`。

## 反馈格式

建议按以下格式反馈，便于我更新 spec 和继续修复：

```text
环境：Windows 版本 / GPU / gcc 版本
脚本：build_xxx.bat
结果：通过 / 失败 / 跳过
现象：窗口、画面、声音、输入的实际表现
日志：复制关键错误行
```

XUI 验证程序退出时会打印 summary：

- `xui layout summary`：关注 `size`、`content` 和 `resize` 是否与最后窗口尺寸、布局显示一致。
- `xui input summary`：关注 `normal_cursor`、`normal_select`、`normal_scroll`、`password_len`、`readonly`、`disabled`。
- `xui textedit summary`：关注 `cursor`、`select`、`scroll`、`lines`、`visual`、`undo`、`redo`、`wrap`。

XUI Windows 人工验证结果统一记录到 `dev/docs/XUI Windows人工验证记录.md`。

平台后端相关结果应同步写入 `docs/平台后端验证结果.md`。该文件是关闭 Linux/macOS/Android/iOS/Web/板卡 EGL 等平台任务的依据。

## spec 关闭规则

只有当第一到第六组中当前机器可执行项目完成验证，且失败项已修复或明确转入后续任务后，才能关闭 `XGE_V2_SPEC.md` 中的“完成人工冒烟测试”。
