# XGE 人工冒烟测试流程

本文档用于集中人工验证当前 XGE 第一阶段 MVP。自动测试只能覆盖 API 行为和无窗口路径，窗口显示、输入法、音频设备、GPU 驱动、小程序和板卡 Linux 仍需要人工观察。

## 测试前准备

- 在 Windows 上确认 `gcc` 位于 `PATH`。
- 从仓库根目录执行脚本。
- 每个示例启动后观察 10 到 30 秒，确认没有崩溃、黑屏、明显卡死或异常闪烁。
- 如果示例需要交互，至少执行一次对应输入。
- 如果某项因为本机缺少设备无法验证，在反馈中标记为 `跳过`，不要视为失败。

## Windows 手工综合验证

当前 Windows 收敛优先使用综合验证程序覆盖窗口、渲染、输入、IME、gamepad 模拟和压力图层：

```bat
build_manual_validation_exe.bat
build\xge_manual_validation.exe --stress 20000 --novsync
```

通过标准：

- 顶部仪表盘文字清晰可见，压力背景不会遮挡文字、准星或三块视觉测试区。
- 棋盘纹理、shape 面板、移动圆、鼠标准星均显示正常。
- 鼠标移动会更新 `mouse` 与 `delta`；左键、右键、滚轮计数会增加。
- 左键和空格会触发左侧蓝色矩形弹动；滚轮会触发黄色圆变大后回落。
- 英文输入和中文 IME 上屏会增加 `text events`，`last U+` 会更新。
- 按 `G` 可切换 `gamepad0 connected`，连接状态行应变为高亮绿色。
- 窗口缩放、最小化/恢复、运行 3 到 5 分钟无崩溃、黑屏、残影或明显卡死。
- 按 `ESC` 正常退出，并打印 `manual summary`。

压力基线参考：

```bat
build\xge_manual_validation.exe --bench 10 --stress 20000 --novsync
```

当前 Windows + AMD Radeon RX 6600 基线：`stress=20000` 基本流畅，`draw` 约 130，`batch` 约 6。该数值用于回归对比，不作为其他 GPU 的硬性门槛。

## 第一组：构建与核心回归

```bat
build_test.bat
build_dll.bat
build_exe.bat
```

通过标准：

- `build_test.bat` 退出码为 0。
- `build_dll.bat` 能生成 DLL。
- `build_exe.bat` 能生成并启动最小窗口。
- 最小窗口能清屏、刷新、正常关闭。

## 第二组：基础绘制

```bat
build_shape_exe.bat
build_texture_exe.bat
build_text_exe.bat
build_sprite_batch_exe.bat
```

通过标准：

- shape 示例能看到点、线、矩形、圆、弧线、三角形或多边形。
- texture 示例能加载并显示纹理。
- text 示例能显示英文和中文文本，文本没有明显错位。
- sprite batch 示例能显示多 sprite，窗口运行期间无崩溃。

## 第三组：渲染管线

```bat
build_render_target_exe.bat
build_shader_material_exe.bat
build_shader_distortion_exe.bat
build_perspective_quad_exe.bat
build_isometric_depth_exe.bat
```

通过标准：

- render target 示例能完成离屏绘制再显示到窗口。
- shader/material 示例能显示材质效果。
- distortion 示例能看到扭曲效果。
- perspective quad 示例能看到透视或 2.5D 变形。
- isometric depth 示例中遮挡顺序符合画面预期。

## 第四组：输入与场景

```bat
build_input_exe.bat
build_text_input_exe.bat
build_touch_exe.bat
build_gamepad_exe.bat
build_scene_exe.bat
build_app_mode_exe.bat
```

通过标准：

- keyboard/mouse 示例能响应按键、鼠标移动、点击、滚轮。
- text input 示例能输入英文，系统输入法能提交中文文本。
- touch 示例在触控设备上能响应触点；无触控设备可跳过。
- gamepad 示例在手柄连接时能看到按键或摇杆状态；无手柄可跳过。
- scene 示例能进入、返回或切换场景。
- app-mode 示例能在手动刷新模式下正确更新画面。

## 第五组：资源、音频、调试与 XUI

```bat
build_async_exe.bat
build_audio_exe.bat
build_debug_overlay_exe.bat
build_xui_incubation_exe.bat
build_xui_bridge_exe.bat
```

通过标准：

- async 示例能显示 fallback 资源并在加载完成后切换到目标资源。
- audio 示例能播放 sound/music，音量、循环或 3D 位置接口没有崩溃。
- debug overlay 示例能显示帧统计或调试信息。
- XUI incubation 示例能显示基础控件。
- XUI bridge 示例能把输入事件传递到 XUI，并能触发控件状态变化。

## 第六组：离屏与小程序

```bat
build_miniprogram.bat
```

离屏示例当前位于 `examples/offscreen`，若本机已具备对应 EGL/OpenGL 环境，可按示例目录说明单独执行。

通过标准：

- 小程序构建脚本能生成或复制 hello scaffold 需要的文件。
- offscreen 示例在支持 EGL 的环境下能创建 pbuffer 或报告 fallback 状态。
- readback 数据非空；无 EGL 环境时 fallback 标记应清晰可见。

## 第七组：板卡 Linux

板卡 Linux 验证按 `docs/板卡Linux冒烟测试计划.md` 执行。此项依赖真实 RK3566 或同级 GLES3/EGL 设备，不纳入 Windows 本机通过标准。

## 反馈格式

建议按以下格式反馈，便于我更新 spec 和继续修复：

```text
环境：Windows 版本 / GPU / gcc 版本
脚本：build_xxx.bat
结果：通过 / 失败 / 跳过
现象：窗口、画面、声音、输入的实际表现
日志：复制关键错误行
```

平台后端相关结果应同步写入 `docs/平台后端验证结果.md`。该文件是关闭 Linux/macOS/Android/iOS/Web/板卡 EGL 等平台任务的依据。

## spec 关闭规则

只有当第一到第六组中当前机器可执行项目完成验证，且失败项已修复或明确转入后续任务后，才能关闭 `XGE_V2_SPEC.md` 中的“完成人工冒烟测试”。
