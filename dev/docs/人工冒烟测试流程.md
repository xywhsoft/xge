# XGE 人工冒烟测试流程

本文档用于集中人工验证当前 XGE 第一阶段 MVP。自动测试只能覆盖 API 行为和无窗口路径，窗口显示、输入法、音频设备、GPU 驱动、小程序和板卡 Linux 仍需要人工观察。

## 测试前准备

- 在 Windows 上确认 `gcc` 位于 `PATH`。
- 从仓库根目录执行脚本。
- 每个示例启动后观察 10 到 30 秒，确认没有崩溃、黑屏、明显卡死或异常闪烁。
- 如果示例需要交互，至少执行一次对应输入。
- 如果某项因为本机缺少设备无法验证，在反馈中标记为 `跳过`，不要视为失败。

## Windows 手工综合验证

当前 Windows 收敛优先使用综合验证程序覆盖窗口、渲染、输入、IME、gamepad 模拟和压力图层：

```bat
examples\manual_validation\build.bat
build\xge_manual_validation.exe --stress 20000 --novsync
```

通过标准：

- 顶部仪表盘文字清晰可见，压力背景不会遮挡文字、准星或三块视觉测试区。
- 棋盘纹理、shape 面板、移动圆、鼠标准星均显示正常。
- 鼠标移动会更新 `mouse` 与 `delta`；左键、右键、滚轮计数会增加。
- 左键和空格会触发左侧蓝色矩形弹动；滚轮会触发黄色圆变大后回落。
- 英文输入和中文 IME 上屏会增加 `text events`，`last U+` 会更新。
- 按 `G` 可切换 `gamepad0 connected`，连接状态行应变为高亮绿色。
- 窗口缩放、最小化/恢复、运行 3 到 5 分钟无崩溃、黑屏、残影或明显卡死。
- 按 `ESC` 正常退出，并打印 `manual summary`。

压力基线参考：

```bat
build\xge_manual_validation.exe --bench 10 --stress 20000 --novsync
```

当前 Windows + AMD Radeon RX 6600 基线：`stress=20000` 基本流畅，`draw` 约 130，`batch` 约 6。该数值用于回归对比，不作为其他 GPU 的硬性门槛。

```text

脚本：build_xxx.bat
结果：通过 / 失败 / 跳过
现象：窗口、画面、声音、输入的实际表现
日志：复制关键错误行
```

- `xui textedit summary`：关注 `cursor`、`select`、`scroll`、`lines`、`visual`、`undo`、`redo`、`wrap`。
- `xui-text-edit-lab final-summary`：关注 `input`、`edit`、`cursor`、`ro`、`clipmenu`、`undo`、`wrap`、`ime`、`menus`。
- `xui-text-undo-stack-lab final-summary`：关注 `empty`、`direct`、`selection`、`keyboard`、`clear`、`undo`、`redo`。
- `xui-menu-owner-lab final-summary`：关注 `init`、`config`、`clamp`、`disabled`、`select`、`close`、`selected`、`cb`。
- `xui-popup-owner-lab final-summary`：关注 `init`、`open`、`owner`、`autoclose`、`proc`、`bg`、`close`。
- `xui-tooltip-owner-lab final-summary`：关注 `init`、`open`、`offset`、`enabled`、`text`、`direct`、`legacy`。
