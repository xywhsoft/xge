# XGE 单头文件使用说明

XGE 仓库主源码采用模块化结构，单头文件只作为发布和第三方集成产物。

## 当前状态

当前阶段：

- 开发入口是根目录 `xge.h` + `xge.c`。
- 实现源码位于 `src/`。
- 第三方依赖位于 `lib/`。
- 单头文件目录为 `singlehead/`。
- 单头文件生成器尚未正式实现。

因此当前开发、测试和示例都应使用：

```c
#include "xge.h"
```

并在编译时加入：

```text
xge.c
```

## 设计目标

单头文件版本的目标是降低集成成本，而不是改变仓库开发方式。

最终发布形态计划为：

- `singlehead/xge_single.h`：发布用单头文件。
- `singlehead/gen_singlehead.*`：单头文件生成器。
- `singlehead/test_singlehead.c`：单头文件集成测试。

## 使用方式

生成器完成后，外部项目可采用类似方式集成：

```c
#define XGE_IMPL
#include "xge_single.h"
```

或在一个独立编译单元中启用实现：

```c
// xge_impl.c
#define XGE_IMPL
#include "xge_single.h"
```

其他源码文件只包含声明：

```c
#include "xge_single.h"
```

具体宏名会在生成器落地时最终确认。当前仓库源码仍以 `xge.c` 为实现入口。

## 维护规则

- 不允许把大量实现代码直接写入根目录 `xge.h`。
- `xge.h` 只保留公开类型、常量和函数声明。
- 实现按功能模块维护在 `src/`。
- `singlehead/` 生成器负责按固定顺序拼接公开头和实现模块。
- 单头文件生成结果不作为人工维护源文件。
- 修改公开 API 时，先更新 `xge.h`，再更新实现和测试。

## 生成顺序要求

单头文件生成器必须遵守 `src/xge_impl.c` 的聚合顺序。当前顺序大致为：

```c
#include "xge_core.c"
#include "xge_resource.c"
#include "xge_audio.c"
#include "xge_font.c"
#include "xge_texture.c"
#include "xge_render.c"
#include "xge_shape.c"
#include "xge_mesh.c"
#include "xge_sprite.c"
#include "xge_material.c"
#include "xge_render_target.c"
#include "xge_buffer.c"
#include "xge_async.c"
#include "xge_input.c"
#include "xge_xui_host.c"
#include "xge_xui_layout.c"
#include "xge_xui_core.c"
#include "xge_xui_text.c"
#include "xge_xui_controls.c"
#include "xge_xui_controls_extra.c"
#include "xge_xui_scroll.c"
#include "xge_xui.c"
```

如果模块之间的内部依赖发生变化，必须先调整 `src/xge_impl.c`，再调整生成器。

## 测试要求

生成器完成后，至少需要保留两条测试路径：

- 常规源码拆分版：`build_test.*`
- 单头文件版：`singlehead/test_singlehead.c`

两者应覆盖相同核心 API，确保单头文件产物没有漏掉宏、类型、静态函数或实现模块。

## 与 DLL/EXE 的关系

单头文件是集成形态，不取代 DLL/EXE 构建脚本。

- `build_dll.*` 用于验证动态库输出。
- `build_exe.*` 和示例脚本用于验证常规源码构建。
- 单头文件测试只验证发布产物能独立被第三方项目包含和编译。
