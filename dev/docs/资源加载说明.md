# XGE 资源加载说明

本文档记录 XGE V2 当前资源加载路径、资源协议、同步/异步 API、fallback 资源和后续扩展方向。

## 资源根目录

XGE 默认以 xrt 提供的 exe path 作为相对路径根目录。

普通相对路径：

```c
xgeTextureLoad(&texture, "assets/player.png");
xgeFontLoad(&font, "assets/ui.ttf", 18.0f);
```

会从程序根路径下解析。

## URI 协议

当前资源入口为：

```c
int xgeResourceLoad(const char* sURI, xge_resource_t* pResource);
```

支持三类路径：

- 普通相对/绝对文件路径。
- `file://...`
- `res://...`

当前 `file://` 和 `res://` 在没有自定义 provider 时都会剥离协议头，然后按文件路径读取。

示例：

```c
xgeResourceLoad("assets/a.png", &res);
xgeResourceLoad("file://assets/a.png", &res);
xgeResourceLoad("res://assets/a.png", &res);
```

## 自定义 Resource Provider

资源 provider 用于接入压缩包、虚拟文件系统、网络缓存或小程序资源桥接。

结构：

```c
typedef struct xge_resource_provider_t {
	const char* sScheme;
	int (*load)(const char* sURI, void** ppData, int* pSize, void* pUser);
	void (*free)(void* pData, void* pUser);
	void* pUser;
} xge_resource_provider_t;
```

注册：

```c
xgeResourceProviderAdd(&provider);
```

清空：

```c
xgeResourceProviderClear();
```

Provider 数量当前上限为 `XGE_RESOURCE_PROVIDER_MAX`。

Provider 返回的数据所有权由 `xge_resource_t` 持有，调用者必须使用：

```c
xgeResourceFree(&res);
```

释放。

## Memory Resource

内存资源可通过：

```c
xgeResourceLoadMemory(data, size, &res);
```

复制一份数据到 XGE 管理的资源对象。释放同样使用 `xgeResourceFree()`。

## Image 与 Texture

图片加载：

```c
xgeImageLoad(&image, "assets/a.png");
xgeImageLoadEx(&image, "assets/a.png", XGE_IMAGE_PREMULTIPLIED);
xgeImageLoadMemory(&image, data, size);
xgeImageLoadMemoryEx(&image, data, size, XGE_IMAGE_STRAIGHT_ALPHA);
```

纹理加载：

```c
xgeTextureLoad(&texture, "assets/a.png");
xgeTextureLoadEx(&texture, "assets/a.png", XGE_IMAGE_PREMULTIPLIED);
xgeTextureLoadMemory(&texture, data, size);
xgeTextureLoadMemoryEx(&texture, data, size, XGE_IMAGE_PREMULTIPLIED);
```

默认使用 premultiplied alpha。需要直通 alpha 时使用 `XGE_IMAGE_STRAIGHT_ALPHA`。

GPU 纹理也保留 CPU shadow，用于慢路径 readback 和后台加载后的上传队列。

## Texture Fallback

纹理 fallback 用于微端、异步加载或资源缺失场景：

```c
xgeTextureFallbackSetRGBA(width, height, pixels);
xgeTextureFallbackGet(&texture);
xgeTextureFallbackClear();
```

当 `xgeTextureLoadEx()` 或 `xgeTextureLoadMemoryEx()` 失败时，如果 fallback texture 已设置，会自动生成一份 fallback texture，并设置 `XGE_TEXTURE_FALLBACK` 标志。

## GPU Upload Queue

纹理上传队列 API：

```c
xgeTextureUploadQueue(&texture);
xgeTextureUploadFlush();
```

用途：

- 允许资源加载阶段先准备 CPU shadow。
- 在 GL context 所在线程 flush 到 GPU。
- 当前 `xgeFrame()`、Sokol frame 和 `xgeFlush()` 都会调用 upload flush。

## Font 与 XRF

TTF 字体：

```c
xgeFontLoad(&font, "assets/ui.ttf", 18.0f);
xgeFontLoadMemory(&font, data, size, 18.0f);
```

XRF 点阵字体：

```c
xgeFontLoadXRF(&font, "assets/ui.xrf");
xgeFontLoadXRFMemory(&font, data, size);
```

Font fallback：

```c
xgeFontFallbackSet("assets/fallback.ttf", 18.0f);
xgeFontFallbackSetMemory(data, size, 18.0f);
xgeFontFallbackGet(&font, 18.0f);
xgeFontFallbackClear();
```

普通字体对象也可设置自己的 fallback：

```c
xgeFontSetFallback(&font, &fallbackFont);
```

## Sound Fallback

音效 fallback：

```c
xgeSoundFallbackSet("assets/fallback.wav");
xgeSoundFallbackGet(&sound);
xgeSoundFallbackClear();
```

当前 fallback 主要用于 `XGE_AUDIO_SOUND`，不自动覆盖 music/stream。

## 异步加载 API

异步请求对象：

```c
xge_async_request_t req;
xgeAsyncRequestInit(&req);
```

加载接口：

```c
xgeAsyncImageLoad(&req, &image, "assets/a.png", XGE_IMAGE_PREMULTIPLIED, callback, user);
xgeAsyncTextureLoad(&req, &texture, "assets/a.png", XGE_IMAGE_PREMULTIPLIED, callback, user);
xgeAsyncFontLoad(&req, &font, "assets/ui.ttf", 18.0f, callback, user);
xgeAsyncSoundLoad(&req, &sound, "assets/click.wav", callback, user);
```

状态：

- `XGE_ASYNC_PENDING`
- `XGE_ASYNC_LOADING`
- `XGE_ASYNC_READY`
- `XGE_ASYNC_FAILED`
- `XGE_ASYNC_CANCELLED`

当前实现是同线程立即完成语义：调用加载函数后会立刻执行实际加载并完成 request 状态。后续接入 xrt 线程/任务系统后，API 形态保持不变，内部实现改为真正后台加载。

取消：

```c
xgeAsyncRequestCancel(&req);
```

释放：

```c
xgeAsyncRequestFree(&req);
```

## 小程序与 xpack 扩展点

xpack 不进入 XGE 内核。未来可通过 resource provider 接入：

```text
res://pack/path.png
xpack://index/path.png
```

小程序资源加载也应通过 provider 或独立 platform bridge 接入，不应把平台资源逻辑散落到 texture/font/audio 模块。

## 错误与所有权

常见返回：

- `XGE_ERROR_INVALID_ARGUMENT`：参数错误。
- `XGE_ERROR_FILE_NOT_FOUND`：文件不存在。
- `XGE_ERROR_UNSUPPORTED`：协议或功能不支持。
- `XGE_ERROR_RESOURCE_FAILED`：资源解析失败。
- `XGE_ERROR_OUT_OF_MEMORY`：内存不足。

所有 `xgeResourceLoad()` 或 `xgeResourceLoadMemory()` 得到的 `xge_resource_t` 都必须调用 `xgeResourceFree()`。

图片、纹理、字体、音频对象分别使用对应释放函数：

```c
xgeImageFree(&image);
xgeTextureFree(&texture);
xgeFontFree(&font);
xgeSoundFree(&sound);
```
