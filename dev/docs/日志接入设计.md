# XGE 日志接入设计

本文档用于约束 XGE 接入 xrt logger 的方式。当前 `lib/xrt/xrt.h` 已提供 `xlogCreate`、`xlogDefault`、`xlogWrite`、`xlogFlush` 等 logger API，XGE 已通过薄适配层接入。

## 当前状态

当前可用能力：

- `xrtSetError` 可记录最后错误文本。
- shader compile/link 失败时已经把 GL info log 写入 `xrtSetError`。
- debug stats 和 caps dump 已能通过 XGE API 查询。

当前已实现能力：

- `xgeLogSetLevel`
- `xgeLogGetLevel`
- `xgeLogWrite`
- `xgeLogFlush`
- 内部 `__xgeLogFormat` / `__xgeLogError` 适配层
- core/platform/graphics/resource/audio 关键失败路径日志

## 目标

XGE 日志只作为诊断系统，不进入性能关键路径。目标是：

- debug 模式提供足够错误上下文。
- release 模式默认低开销。
- 所有日志最终走 xrt logger。
- XGE 内部只依赖薄适配层，不在各模块直接绑定 xrt logger 具体 API。
- 日志调用不能导致渲染或音频实时线程阻塞。

## 日志级别

建议保留以下级别：

- `trace`：极细粒度调试，默认关闭。
- `debug`：开发期状态检查。
- `info`：关键生命周期信息，例如后端初始化成功。
- `warn`：可恢复问题，例如 fallback resource 被使用。
- `error`：操作失败，但进程仍可继续。
- `fatal`：无法继续运行的严重错误。

## 模块 tag

建议使用固定字符串 tag：

- `core`
- `platform`
- `graphics`
- `render`
- `resource`
- `audio`
- `font`
- `xrf`
- `xui`
- `miniprogram`
- `offscreen`
- `debug`

tag 应保持短小稳定，便于外部过滤。

## XGE 内部适配层

XGE 内部模块使用适配函数，不直接散落 xrt logger 调用：

```c
static void __xgeLogFormat(int iLevel, const char* sTag, const char* sFormat, ...);
static void __xgeLogError(const char* sTag, const char* sMessage);
```

约束：

- 内部函数放在 core/debug 相关源码中。
- 实时音频回调和未来 render thread 内不做格式化字符串。

## 与 xrtSetError 的关系

`xrtSetError` 继续表示“最近一次可查询错误”，不等同于日志系统：

- 返回错误码的函数可同时设置 `xrtSetError`。
- warning/info/debug 不写 `xrtSetError`。
- shader compile/link 错误继续设置 `xrtSetError`，同时写 error log。
- 文件加载失败、资源协议失败、后端初始化失败应设置 `xrtSetError`。

## Debug Overlay

Debug overlay 不直接显示完整日志，只显示摘要：

- 最近错误文本。
- warn/error 计数。
- 当前 backend/caps。
- frame stats。

完整日志由 xrt logger sink 负责，例如控制台、文件或平台调试输出。

## 线程策略

第一版接入时：

- 主线程可直接调用适配层。
- 资源线程可直接调用适配层，但 xrt logger sink 必须线程安全。
- render thread 只提交短消息或计数，不执行复杂格式化。
- 音频实时回调禁止写日志；只能设置原子状态，由主线程汇总。

## spec 关闭条件

`XGE_V2_SPEC.md` 中两项 logger 任务关闭条件：

- xrt 提供稳定 logger API 或 XGE vendored xrt 已包含该 API。（ 已完成 ）
- XGE 内部增加统一日志适配层。（ 已完成 ）
- 至少 core/platform/graphics/resource/audio 五个模块的关键失败路径接入日志。（ 已完成 ）
- shader compile/link 错误同时写入日志和 `xrtSetError`。（ 已完成 ）
- debug 模式下有测试或示例能验证日志路径不会崩溃。（ 已完成 ）

## 当前最终约定

截至当前文档工程阶段，XGE 日志路线按以下规则执行：

- 对外 API 保持 `xgeLogSetLevel`、`xgeLogGetLevel`、`xgeLogWrite`、`xgeLogFlush`。
- 内部模块不直接散落 xrt logger 调用，统一经过 `__xgeLogFormat` / `__xgeLogError`。
- `xrtSetError` 仍表示最近错误文本，日志表示事件流，两者职责不同。
- 音频实时回调不写日志。
- render thread 不做复杂格式化；必要信息由提交侧或主线程汇总。
- debug overlay 只显示摘要，不承担完整日志查看职责。

如果 xrt logger 后续扩展 sink、异步写入或结构化日志，XGE 只调整适配层，不改公开日志 API。
