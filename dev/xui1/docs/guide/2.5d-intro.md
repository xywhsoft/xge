# 2.5D 入门

本教程说明 XGE 如何在纯 2D 引擎里提供有限的 2.5D 能力。

[返回教程索引](README.md) | [Graphics API](../api/graphics.md) | [2.5D 范例](../case/perspective-quad.md)

## 定位

XGE 不把第一版做成 3D 引擎。2.5D API 只解决少量 2D 项目常见需求：

- 透视卡牌和面板。
- 图片四边形扭曲。
- 伪 3D 地面、贴花和阴影。
- 少量 z 排序元素。
- 特效 mesh。

如果项目需要模型、骨骼、光照、材质系统和场景剔除，应使用专门的 3D 引擎。

## 透视 Camera

```c
xge_camera_t camera;

camera = xgeCameraPerspective((float)xgeGetWidth(), (float)xgeGetHeight(), 45.0f, 0.1f, 100.0f);
xgeCameraSet(&camera);
xgeDepthTestSet(1);
```

2.5D 绘制结束后恢复普通 2D 状态：

```c
xgeDepthTestSet(0);
xgeCameraSet(NULL);
```

## 3D Quad

```c
xge_vertex_t vertices[4];
uint32_t white;

memset(vertices, 0, sizeof(vertices));
white = xgeColorRGBA(255, 255, 255, 255);

vertices[0].fX = -1.0f; vertices[0].fY = -1.0f; vertices[0].fZ = -4.0f; vertices[0].fW = 1.0f; vertices[0].fU = 0.0f; vertices[0].fV = 1.0f; vertices[0].iColor = white;
vertices[1].fX =  1.0f; vertices[1].fY = -1.0f; vertices[1].fZ = -4.0f; vertices[1].fW = 1.0f; vertices[1].fU = 1.0f; vertices[1].fV = 1.0f; vertices[1].iColor = white;
vertices[2].fX =  1.2f; vertices[2].fY =  1.0f; vertices[2].fZ = -5.0f; vertices[2].fW = 1.0f; vertices[2].fU = 1.0f; vertices[2].fV = 0.0f; vertices[2].iColor = white;
vertices[3].fX = -1.2f; vertices[3].fY =  1.0f; vertices[3].fZ = -5.0f; vertices[3].fW = 1.0f; vertices[3].fU = 0.0f; vertices[3].fV = 0.0f; vertices[3].iColor = white;

xgeDrawQuad3D(&texture, vertices, 0);
```

如果顶点数量较多，用 mesh：

```c
xgeMeshCreate(&mesh, vertices, vertex_count, indices, index_count, 0);
xgeMeshDraw(&mesh, &texture, 0);
```

## 与 2D 绘制混用

建议顺序：

```text
2D 背景
2.5D pass
2D 前景
XUI / Debug overlay
```

2.5D pass 内部集中启用 depth test，结束后立即恢复，避免 UI 被 z 值影响。

## 常见错误

不要把所有 sprite 都改成 3D quad。普通 sprite 走 2D batch 更快。

不要在 UI 绘制前忘记关闭 depth test。

不要在 2.5D 中依赖复杂 3D 约定。当前目标是少量高级 2D 效果，而不是完整 3D 管线。
