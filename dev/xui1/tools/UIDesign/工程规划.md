# XUI Designer 工程规划

本文档记录 UIDesign / XUI Designer 的工程级决策。后续实现、目录结构、构建脚本和控件编辑管理层都以本文为准。

## 基本定位

- 工具标题：`XUI Designer`
- 工程目录：`tools/UIDesign`
- 工程形态：纯 C 工具，基于 XGE/XUI 原生实现。
- UI 风格：接近 Visual Basic / Visual Studio Form Designer。
- 运行模式：工具类 APP 模式，默认按需刷新，空闲不持续重绘。
- 构建方式：沿用 `tools/mapedit` 的工程布局和 `build.bat` 风格。
- 主界面框架：根布局直接采用 `菜单 -> 工具栏 -> DockPanel -> 状态栏`。DockPanel 是工作区基础框架，承载 Toolbox、Structure、Form Designer、Project Explorer、Properties、Actions、Output 等窗口；不在 DockPanel 外再做额外三栏/底栏布局。

## 工程与页面保存格式

XUI Designer 支持一个工程组织多个页面。

推荐工程结构：

```text
MyUiProject/
  project.xson
  pages/
    main.xson
    settings.xson
    dialog_about.xson
  styles/
    theme.xson
    controls.xson
  res/
    images/
    icons/
    fonts/
  assets/
```

`project.xson` 保存：

- 工程名称。
- 页面列表。
- 默认启动页面。
- 多平台预览尺寸配置。
- 全局资源索引。
- 全局 action/event registry。
- 工程级 styles/tokens 引用。
- 编辑器布局状态。

页面保存为独立 `.xson`，每个页面可单独加载为 XUI Page。

## XSON 写回策略

第一版采用格式化重写，不追求保留原始注释、空白、字段顺序和用户手写格式。

规则：

- `UIDocument` 是设计时唯一真源。
- 保存时从 `UIDocument` 生成规范 XSON。
- 导入已有 XSON 后允许规范化字段顺序。
- 对用户不可表达或当前设计器不支持的字段，应保留到 `unknownFields`，除非明确提示用户会丢失。
- 保存前必须做 schema validation。

## 第一阶段控件范围

MVP 先实现：

- 基础控件：`label`、`button`、`input`、`image`、`separator`、`checkbox`、`radio`、`toggle`、`slider`、`progress`。
- 容器布局：`panel`、`row`、`column`、`stack`、`grid`、`dock`、`absolute`、`scrollView`。
- 集合控件：`comboBox`、`listView`、`treeView`、`tabs`。
- 数据控件：`propertyGrid`。
- 工作台控件：`dockLayout`，第一版即实现完整专用编辑器，用来验证控件编辑器扩展机制。

暂缓但预留 schema：

- `tableView`、`tableGrid`。
- `menu`、`menuBar`、`toolbar`、`statusBar`。
- `window`、`popup`、`msgBox`、`inputBox`。
- `colorPicker`、`datePicker`、`numericInput`、`textEdit`。
- `timelineView`。

## 复杂控件策略

复杂控件第一版不是只给基础属性，而是至少选 `dockLayout` 做完整专用编辑器。

目标：

- 验证 `DesignAction` 多入口机制。
- 验证专用 editor window / docked tool window。
- 验证复杂结构模型写回 XSON。
- 验证运行时预览和设计时文档同步。
- 验证 undo/redo 可以覆盖复杂控件操作。

DockLayout 专用编辑器第一版包含：

- DockWindow 管理。
- Region 比例和可见性管理。
- 初始停靠位置、floating rect、hidden/autoHide 状态。
- 窗口 client slot 编辑入口。
- 设计时保存/加载 dock state。
- 画布停靠指示器和拖动预览。

## 事件注册与分发机制

设计时和运行时事件不直接绑定具体语言函数。XUI Designer 维护一套后端无关的 action/event registry，后续可接任意语言后端。

### 工程级 Action Registry

```json
{
  "actions": [
    {
      "id": "app.refresh",
      "name": "Refresh",
      "backend": "native",
      "entry": "AppRefresh",
      "payloadSchema": {}
    }
  ]
}
```

页面控件只引用 action id：

```json
{
  "type": "button",
  "id": "refreshButton",
  "text": "Refresh",
  "onClick": "app.refresh"
}
```

### 运行时分发

设计器生成或运行预览时，把 XSON 事件名绑定到统一 dispatcher：

```c
typedef struct uid_event_t {
    const char* actionId;
    const char* pageId;
    const char* nodeId;
    const char* eventName;
    xvalue payload;
} uid_event_t;

typedef int (*uid_backend_dispatch_proc)(const uid_event_t* event, void* user);
```

后端适配层负责把 `uid_event_t` 转给：

- C 函数。
- 脚本 VM。
- IPC / WebSocket。
- 其他语言插件。

XUI Designer 只管理 action id、参数 schema、示例 payload 和绑定关系，不把 UI 文件绑定死到某种语言。

## 资源管理

导入资源时复制到当前工程目录的 `res/` 或 `assets/`，由工程统一管理。

规则：

- 图片、图标、字体、atlas 等外部资源进入工程目录。
- 页面 XSON 使用相对路径或工程资源 URI。
- 避免保存绝对路径。
- 资源面板提供重命名、替换、查找引用、删除未使用资源。
- 导入重名资源时生成唯一文件名或提示覆盖。
- Button icon、Image texture、Menu icon、字体 token 都通过资源选择器选择。

## 样式系统

样式系统使用 XValue 的继承机制，和 XUI XSON 当前 style/token 方案保持一致。

第一版支持：

- style 表。
- `@parent` 继承。
- inline override。
- color/spacing/font token。
- 当前节点最终样式预览。
- 从 inline 抽取为命名 style。

不做：

- 脚本表达式。
- 运行时动态样式语言。
- 保留手写格式 round-trip。

## 多平台预览

预览目标需要考虑多平台尺寸配置。

工程级保存 viewport presets：

```json
{
  "previewTargets": [
    { "id": "desktop_1366", "name": "Desktop 1366x768", "width": 1366, "height": 768, "dipScale": 1.0 },
    { "id": "phone_portrait", "name": "Phone Portrait", "width": 390, "height": 844, "dipScale": 2.0 },
    { "id": "tablet_landscape", "name": "Tablet Landscape", "width": 1024, "height": 768, "dipScale": 1.5 }
  ]
}
```

设计器能力：

- 画布切换预设尺寸。
- 显示 safe area。
- 模拟 DIP scale。
- 快速横竖屏切换。
- 保存每个页面的默认预览目标。
- 校验固定像素布局在目标尺寸下是否溢出。

## Undo / Redo

规则：

- 属性表单次提交生成一个 command。
- 拖拽 resize/move 从 mouse down 到 mouse up 合并为一个 command。
- 画布连续拖放过程中只刷新预览，不持续入栈。
- 文本编辑整体合并为一个 command，即进入编辑到提交/失焦/确认之间合并。
- 集合编辑器的新增、删除、重排、批量导入分别是 command。
- DockLayout 复杂操作可保存 before/after dock state 作为一个 command。
- 保存文件不清空 undo stack，但记录 clean revision。

Command 应作用于 `UIDocument`，不是直接作用于 runtime widget。

## 按需刷新

XUI Designer 采用 `XGE_INIT_ON_DEMAND`。

必要封装：

```c
void UIDesignInvalidate(uid_app_t* app, int reason);
void UIDesignInvalidateNode(uid_app_t* app, uid_node_id nodeId, int reason);
void UIDesignInvalidateInspector(uid_app_t* app);
void UIDesignBeginContinuousRefresh(uid_app_t* app, int reason);
void UIDesignEndContinuousRefresh(uid_app_t* app, int reason);
```

连续刷新只在这些场景开启：

- 画布拖拽。
- Dock indicator 预览。
- 动画 UI 运行预览。
- 弹层动画或需要连续 hover 动画的控件。

空闲时不更新、不绘制、不 present。
