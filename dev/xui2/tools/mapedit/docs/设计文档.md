# 地图编辑器设计文档

## 目标

地图编辑器中的图块编辑器目标是一站式、功能完整的图块管理器。它不仅用于选择已有图块，还要负责从外部图片素材制作成品图块资源，并将这些资源组织成地图最终使用的图块集。

图块编辑器需要支持：

- 普通静态图块集
- 动画图块
- 自动图块
- 多状态图块
- 多状态自动图块
- 图块通行编辑
- 角色覆盖编辑
- 图块默认数据打标

其中动画、自动图块、多状态图块不是完全互斥的类型，而是可以组合的能力。

## 核心概念

### TileSource 图块素材

`TileSource` 是外部图片素材。

它的职责是：

- 加载任意外部图片
- 按网格切片
- 在图块素材窗口中选择或框选素材区域
- 作为图块制作时的画笔来源

`TileSource` 只用于取样、裁剪和拷贝，不直接进入地图数据，也不直接作为地图渲染资源。

### TileAsset 独立素材图块

`TileAsset` 是 `assets` 目录内的成品独立图块素材。

它可以支持：

- 静态单图块
- 动画图块
- 自动图块
- 多状态图块
- 多状态自动图块

动画图块横向排列帧，多状态图块纵向排列状态。自动图块本身包含一个图块的多种拼接形态，自动图块也可以横向叠加动画帧、纵向叠加状态。

### 资源目录

编辑器生成的成品图块素材统一放在程序目录的 `assets` 下。

```text
assets/
  tilesheets/        静态图块集
  animated_tiles/    动态图块
  autotiles/         自动图块，默认支持动画帧
  state_tiles/       多状态图块，默认支持动画帧
  state_autotiles/   多状态自动图块，默认支持动画帧
```

后面三类默认具备动画能力，因此不再额外拆出 `animated_autotiles`、`animated_state_tiles`、`animated_state_autotiles` 目录。

### TileSheet 静态图块集

`TileSheet` 是 `assets` 目录内的普通静态图块集。

它的特点是：

- 一张图片中包含大量普通静态图块
- 使用 `tileWidth`、`tileHeight`、`tilesPerRow` 计算图块位置
- 适合地面、墙体、装饰等大量不会变化的普通图块

图块 ID 转换为素材坐标的规则：

```text
x = (localTileId % tilesPerRow) * tileWidth
y = (localTileId / tilesPerRow) * tileHeight
```

### TileSet 地图图块集

`TileSet` 是地图最终使用的图块集。

它负责组合：

- 独立素材图块 `TileAsset`
- 静态图块集 `TileSheet`
- 通行数据
- 角色覆盖数据
- 自定义默认数据

地图编辑和地图渲染都应该使用 `TileSet` 的最终图块 ID，而不是直接使用外部图片路径或图片坐标。

## 图块 ID 规则

图块集 ID 规则固定为：

```text
0 = 橡皮，不渲染图块

1 .. independentSlotCount
= 独立素材图块槽位
= TileAsset 引用

independentSlotCount + 1 开始
= 静态图块集 TileSheet 的普通图块
```

例如：

```text
independentSlotCount = 60

0     = erase
1..60 = TileAsset slots
61..  = TileSheet tiles
```

示例结构：

```xson
{
  "id": "world_tileset",
  "tileWidth": 16,
  "tileHeight": 16,
  "independentSlotCount": 60,
  "assetSlots": [
    { "slot": 1, "asset": "water_anim" },
    { "slot": 2, "asset": "tree_season" }
  ],
  "sheets": [
    { "id": "terrain_sheet", "firstTileId": 61 }
  ]
}
```

## 图块制作

图块制作窗口用于将外部图片转换为 `assets` 目录内的成品素材。

它可以选择已有图块文件，也可以从图块素材窗口中取样，把素材画到图块制作的格子里。保存后，外部素材会按照当前制作模式输出为 `TileAsset` 或 `TileSheet`。

图块制作需要支持模式切换：

- 静态图块集制作
- 动画图块制作
- 自动图块制作
- 多状态图块制作
- 多状态自动图块制作

### 静态图块集制作

输出 `TileSheet`。

用于将多个普通静态图块排列到一张图片中。普通静态图块可以很多个图块放在一张图片里。

### 动画图块制作

输出 `TileAsset`。

动画帧横向排列。例如 `16x16` 图块，4 帧动画素材大小为：

```text
64x16
```

渲染时 4 帧图片循环播放。

### 自动图块制作

输出 `TileAsset`。

自动图块第一批优先支持 RPG Maker 类型自动图块。自动图块自身包含一个图块的多种拼接状态，自动图块也应自带动画制作能力。

如果自动图块模板宽高是固定比例，则动画仍然横向排列：

```text
x = frameIndex * patternWidth
y = 0
```

### 多状态图块制作

输出 `TileAsset`。

多状态图块用于通过地图状态修改图块显示样式，例如四季切换、日夜轮替。

多状态纵向排列。例如 `16x16` 图块，4 种状态素材大小为：

```text
16x64
```

如果同时支持动画，例如 4 帧动画和 4 种状态，则素材大小为：

```text
64x64
```

计算规则：

```text
x = frameIndex * tileWidth
y = stateIndex * tileHeight
```

### 多状态自动图块制作

输出 `TileAsset`。

这是自动图块、多状态、动画的组合。自动图块模板横向叠加动画帧，纵向叠加状态。

计算规则：

```text
x = frameIndex * patternWidth
y = stateIndex * patternHeight
```

## 图块素材

图块素材窗口用于加载外部图片。

它的职责是：

- 加载任意图片
- 设置或读取切片规格
- 显示网格
- 选择单个图块或框选区域
- 为图块制作窗口提供取样来源

图块素材不是最终图块资源，它只是制作阶段的来源。

## 图块集工具

图块集工具用于制作地图最终使用的 `TileSet`。

它需要支持：

- 设置独立素材图块槽位数量
- 固定 0 号图块为橡皮
- 将任意 `TileAsset` 插入到独立槽位
- 追加一个或多个 `TileSheet`
- 生成最终图块 ID 映射

例如设置 60 个独立素材图块，则：

```text
1 - 60 号图块 = 独立素材槽位
60 号之后    = 静态图块集素材 ID
```

图块集工具是通行编辑、角色覆盖、图块打标的基础。

## 通行编辑

通行编辑针对 `TileSet` 的最终图块 ID，而不是针对外部素材。

通行数据使用 `byte`：

```text
uint8 passage
```

8 个 bit 分别表示 8 个方向是否允许通行。建议方向定义固定为：

```text
bit0 = 上
bit1 = 右上
bit2 = 右
bit3 = 右下
bit4 = 下
bit5 = 左下
bit6 = 左
bit7 = 左上
```

简单模式：

- 图块不分区域
- 点击后在 `0` 和 `255` 之间切换
- `0` 表示不可通行
- `255` 表示全方向可通行

专家模式：

- 图块分为 9 个区域
- 中心区域快捷切换 `0` / `255`
- 边缘 8 个区域分别切换对应方向 bit

## 角色覆盖

角色覆盖工具用于配置图块遮盖状态，类似 RPG Maker 中草木繁茂处的配置。

该数据同样绑定到 `TileSet` 的最终图块 ID。

## 图块打标

图块打标工具根据配置文件中的 `customData` 信息，为图块配置默认数据。

这部分后续再展开。当前只需要明确：

- 打标目标是 `TileSet` 的最终图块 ID
- 打标字段来自 `setup.xson` 的 `customData`
- 不同数据通道可以有不同数据类型和打标方式

## 实现顺序

建议按以下顺序实现：

1. 在 `map_sdk` 中定义 `TileSource`、`TileAsset`、`TileSheet`、`TileSet` 数据结构。
2. 增加 `tileset.xson` 的加载和保存。
3. 图块素材窗口实现外部图片加载、网格显示和取样选择。
4. 图块制作窗口先实现静态图块集制作。
5. 图块制作窗口继续实现动画图块制作。
6. 增加自动图块制作模式，先支持 RPG Maker 类型。
7. 增加多状态图块和多状态自动图块制作模式。
8. 新增图块集工具，管理独立素材槽位和静态图块集映射。
9. 通行编辑按 `TileSet tileId -> uint8 passage` 实现。
10. 角色覆盖和图块打标基于同一套 `TileSet tileId` 数据模型扩展。
